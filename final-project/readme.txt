/*Χρήσιμες Πληροφορίες*/
Δίαφορα σενάρια εκτέλεσης ελέγχθηκαν με valgrand συνεπώς η μνήμη χειρίζεται και δε χαρίζεται.

Το παιχνίδι μας περιλαμβάνει έλεγχο εισόδου.


Επιπλέον κάθε περιβάλλον χρήσης του ( έστω main()) δεν προορίζεται να το κατευθύνει.Αυτή είναι "υπηρεσία" και αποκλειστική υπευθυνότητα του grid περιβάλλοντος και σε καμία περίπτωση ανοιχτή στο χρήστη.
Αν δεν καλούμασταν να σας παρουσιάσουμε την εργασία για έλεγχο  και βαθμολόγηση δεν θα αφήναμε public συμπεριφορές (συναρτήσεις μέλη) αλλά θα  ορίζαμε μόνο protected & private συμπεριφορές 
φτιάχνωντας ταυτόχρονα το κατάλληλο δίκτυο φιλίας για την ελαχιστική πρόσβαση κλάσεων σε μεθόδους άλλων κλάσεων.
Για να μπορέσετε να το ελέγξετε παίζοντας με την main () τις αφήνουμε public:.


Η srand() στο grid::grid εκούσια δεν αρχικοποιείται αφού σκοπός δεν είναι ενά πραγματικό παιχνίδι απλά μια παρουσίαση και αφού η αναρχικοποίητη δημιουργεί ευνοικό για παρουσίαση χάρτη την αφήσαμε έτσι.
Για ονόματα τεράτων χρησιμοποιήσαμε τη φαντασία μας και συμπληρωματικά πληροφορίες από τον ιστοχώρο με διέυθυνση : https://www.fantasynamegenerators.com/ghost-names.php


/*Συμβάσεις */
Θεωρούμαι ότι τα επίπεδα αρχίζουν από το 0 και πως η συνάρτηση αναβάθμισης επιπέδου είναι γραμμική της εμπειρίας και όχι λογαριθμική όπως στα περισσότερα παιχνίδια χάριν ευκολίας υλοποίησης.
 
Προς διευκόλυνση του παίχτη θεωρούμε οτι τα τέρατα είναι ανόητα και άκαρδα,συνεπώς δεν ελέγχουμαι αν επιτίθονται σε νεκρό ήρωα.Ώστε να μην τους δώσουμε την δυνατότητα να ξαναδιαλέξουν δέκτη επίθεσης.
Αντιθέτως τα αντικείμενα τύπου Ήρωα έχουν αυτό το προνόμιο.
Η υλοποίηση των μεθόδων των κλάσεων δηλαδή λειτουργίες και τυπικοί παράμετροι έχουν ως άξονα την μέγιστη επεκτασιμότητα των κλάσσεων τόσο σαν επι μέρους αρχεία όσο && συνολικά.
Για παράδειγμα μπορούμε να δώσουμε σε 2 άλλους συμφοιτητές μας μόνο τα interfaces μας και τα σχόλια και αυτοί να αναλάβουν να φτιάξουν τον front end του παιχνιδιού χωρίς να χρειαστεί να επικοινωνούμε ενός εμείς παράλληλα εμπλουτίζουμε το back end (ή απλά διαβάζουμε  για sys.pro.)

Οι εκτυπώσεις constructor και destuctor αφαιρέθηκαν όπου κρίθηκε απαραίτητο κατα την τελική επεξεργασία για να μην υπάρχει οπτικός πλεονασμός.
Για να κάνουμε το παιχνίδι ελάχιστα πιο διασκεδαστικό εκτύπωνουμε στο cout αστεία και βοηθητικές για τον παίχτη εκτυπώσεις χειρισμού και πληροφόρησης που ίσως πλεονάζουν συγκριτικά με τα ζητούμενα 
σε καμία περίπτωση όμως δεν κουράζουν.
Δίνουμε τη βάση για παραμετρικά μεγέθη χάρτη,όχι όμως τη συμπεριφορά(εκτυπώσεις στο cout εισαγωγή απο το cin) για να επιταχύνουμε την διαδικασία της παρουσίασης και διόρθωσης χωρίς πολλές-πολλές τετριμμένες λειτουργίες(όπως να σας ζητάγαμε είσοδο για το πόσα θα θέλατε τα βουνά ,τις κοιλάδες,τις αγορές και τα μοναστήρια).


Εντολή μεταγλώττισης:
g++ -o main main.cpp

Εντολή εκτέλεσης:
./main
